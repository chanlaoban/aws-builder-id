📖 操作手册（小白版）

🎯 总体流程（4步）

① 准备域名 → ② 部署邮箱 Worker → ③ 填写配置 → ④ 启动注册
下面每一步都有详细说明，跟着做就行 👇

1

准备一个域名

▼

你需要一个自己的域名来接收 AWS 验证邮件。如果你没有，可以去买一个便宜的。

推荐购买域名的地方：

  • Namesilo（最便宜，约 ¥20/年，含隐私保护）
  • 阿里云 / 腾讯云（国内平台，可以买 .xyz .top 等便宜域名）
  • Cloudflare Registrar（成本价，没有加价）
💡 新手建议买一个 .xyz 或 .top 域名，一年才几块钱。不用买贵的！
2

将域名托管到 Cloudflare（免费）

▼

Cloudflare 提供免费的 DNS 托管和邮件转发服务，是整套方案的核心。

操作步骤：

  1. 打开 https://dash.cloudflare.com 并注册账号（免费）
  2. 登录后点击「添加站点 (Add site)」
  3. 输入你刚买的域名（比如 example.xyz），点击「添加」
  4. 选择 Free（免费） 计划
  5. Cloudflare 会扫描你的 DNS 记录，直接点「继续」
  6. 它会给你两个 Cloudflare 名称服务器（Nameserver），类似：
    daisy.ns.cloudflare.com
    jake.ns.cloudflare.com
  7. 去你买域名的地方，把 DNS 服务器改成上面这两个
  8. 等待 5-30 分钟，Cloudflare 显示「活动中 (Active)」就 OK 了
💡 不懂怎么改 Nameserver？去你买域名的网站搜「修改 DNS 服务器」或「Nameserver 设置」，把原来的替换成 Cloudflare 给的那两个。
3

部署临时邮箱 Worker（最关键的一步）

▼

这个 Worker 负责：创建临时收件箱 → 接收 AWS 验证邮件 → 让注册机取回验证码

我们用的是 cloudflare_temp_email 这个开源项目，一键部署。

操作步骤：

  1. 打开 cloudflare_temp_email 项目
  2. 点 「Deploy with Workers」 按钮（或者在项目里找一键部署链接）
  3. 会跳转到 Cloudflare Dashboard，选择你的账号
  4. 给你的 Worker 起个名字，比如 temp-mail
  5. 点「部署 (Deploy)」，等十几秒就好了
  6. 部署完成后，你会得到一个 Worker URL：
    https://temp-mail.你的子域名.workers.dev
💡 部署完成后，先测试一下。在浏览器打开 https://你的worker地址/api/new_address，应该返回一段 JSON。
4

配置 Cloudflare Email Routing

▼

这一步告诉 Cloudflare：「所有发到 xxx@你的域名 的邮件，都转发给 Worker 处理」

操作步骤：

  1. 登录 Cloudflare Dashboard
  2. 点击你的域名进入管理页面
  3. 左侧菜单找到 「Email」→「Email Routing」
  4. 点击「开始使用 (Get started)」
  5. 选择 Free（免费） 计划
  6. 填写你的邮箱（用于验证域名所有权），收验证码验证
  7. 验证完成后，点击 「Catch-All 地址」 或 「Routing rules」
  8. 添加一条规则：
    - 操作：「发送到 Worker」
    - Worker 选择：你刚刚部署的 Worker（比如 temp-mail）
    - 点击保存
  9. 确保 Catch-All 状态是 「Enabled」
✅ 到这一步，邮箱服务就配好了！你的 Worker URL 和域名就是下面配置页面要填的内容。

📌 记住这两个值，待会要用：

  • Worker URL → 格式如 https://temp-mail.xxx.workers.dev（从第3步得到）
  • 域名 → 你的域名，如 example.xyz
5

在页面填写配置并运行

▼

回到本页面的 「⚙️ 配置」 Tab，填写以下必填项：

字段 填什么 从哪里获取
Worker 地址 ⭐ https://temp-mail.xxx.workers.dev 第3步部署完成后得到的 URL
邮箱域名 ⭐ example.xyz 你买的域名，已托管到 Cloudflare
目标地区 ⭐ 美国 / 德国 / 日本 建议选美国 (USA)

然后按步骤操作：

  1. 点击「🔍 环境检测」Tab → 点「开始检测」，确保所有项都通过
  2. 确认 Chrome ✅、ChromeDriver ✅、IP 纯净度 OK（风险分低于 50）
  3. 点击「🎮 控制台」Tab
  4. 点击「🚀 启动注册」按钮
  5. 等待运行完成，日志会显示注册进度
  6. 注册成功的账号会出现在「📋 账号管理」Tab
💡 第一次运行建议把「无头模式」去掉勾选（关掉），这样能看到浏览器窗口在做什么。调试没问题再勾上。
⚠️ 如果没有配置代理，注册机会用本机 IP。请确保 IP 属地和你选的地区一致（比如选美国就要用美国 IP）

❓ 常见问题

我没有域名怎么办？
去买一个便宜的。推荐 .xyz 或 .top，一年只需 ¥5-¥20。推荐 Namesilo 或 阿里云。
一定要用 Cloudflare 吗？
是的，这套方案依赖 Cloudflare 的 Email Routing（免费）和 Workers（免费额度足够）。如果已经有域名在其他平台，可以免费转入 Cloudflare。
Worker 部署后怎么测试能不能用？
在浏览器打开 https://你的worker地址/api/new_address，如果返回类似 {"address":"xxx@你的域名"} 的 JSON，说明 Worker 正常工作。
环境检测显示 IP 风险高怎么办？
如果 IP 被标记为代理或数据中心，AWS 可能会阻止注册。建议使用住宅代理（家庭宽带 IP）。在配置页勾选「启用代理」，填入住宅代理地址。免费的机房 IP 很容易被 AWS 拦截。
注册失败了，怎么看原因？
去「控制台」Tab 看日志。常见失败原因：① 邮箱 Worker 没配好（验证码收不到）② IP 被 AWS 拦截 ③ 浏览器检测到自动化。可以根据日志中的错误信息排查。
注册一个账号要多久？
通常 1-3 分钟。主要耗时在等待验证邮件（约 30-60 秒）和填写表单。
这个工具安全吗？
所有操作都在你自己的服务器上运行，账号信息保存在本地 accounts.jsonl 文件中，不会上传到任何第三方。
